# Performance Benchmark: 10x 10KB TRN-heavy AFP Files

This benchmark evaluates the performance of the AFP-to-XML conversion across multiple releases, including the first (**v0.1**) and latest (**v12.0**), using a test set of 10 AFP files, each approximately 10KB, consisting primarily of Transparent Data (TRN) control sequences.

## Benchmark Environment
- **Hardware**: Sandbox Environment
- **Java Version**: 21
- **Test Data**: 10 files, ~10KB each, heavy TRN usage.
- **Methodology**: Total time for 10 full iterations (converting all 10 files 10 times).

## Results

1.  **Aggressive I/O (-a) Impact**: For these small 10KB files, the aggressive I/O flag (`-a`) has negligible impact on performance. The overhead of setting up memory-mapped I/O (where applicable) or other aggressive optimizations is often equal to or slightly greater than the simple sequential write for such tiny payloads.
2.  **Massive Performance Leap (v3.0+)**: The introduction of directory processing (`-d`) and later parallel processing (`-p`) significantly improved throughput compared to the manual file-by-file loop required in v0.1 and v1.0.
3.  **Overhead in Later Versions**: There is a noticeable increase in execution time from v7.0 to v12.0. This is attributed to the increased complexity of the parser, more comprehensive validation, and the transition to the Jackson-based `AfpJacksonXmlWriter` as the primary writer.
4.  **TRN Handling**: For TRN-heavy files of this small size (10KB), the overhead of thread orchestration and complex writer logic in newer versions outweighs the benefits for this specific tiny-payload scenario.

## Appendix: Measurement Script

```bash
#!/bin/bash

# Capture current branch to return to it later
INITIAL_BRANCH=$(git rev-parse --abbrev-ref HEAD)
if [ "$INITIAL_BRANCH" = "HEAD" ]; then
    INITIAL_BRANCH=$(git rev-parse HEAD)
fi

# Selected releases to investigate further, including first (v0.1) and last (v12.0)
RELEASES=("v0.1" "v1.0" "v3.0" "v3.4" "v5.3" "v7.0" "v9.11" "v12.0")
TEST_DIR="perf_test"
ITERATIONS=10
OUTPUT_DIR="test_output_bench_10x10"

# Temporary directory to hold test files across checkouts
TEMP_TEST_DIR=$(mktemp -d)
cp "$TEST_DIR"/*.afp "$TEMP_TEST_DIR"

mkdir -p "$OUTPUT_DIR"

echo "| Release | Optimization Flags | Total Time (10 runs of 10 files) | Avg Time per run |"
echo "| :--- | :--- | :--- | :--- |"

# Stash current changes to allow checkout
git stash push --quiet

run_benchmark() {
    local rel=$1
    local flags=$2
    local help_out=$3
    local jar=$4

    # Start timing
    START_TIME=$(date +%s%3N)
    for i in $(seq 1 $ITERATIONS); do
        # Check if -d (directory mode) is supported
        if echo "$help_out" | grep -q -- "-d"; then
             java -jar "$jar" $flags -d "$TEMP_TEST_DIR" "$OUTPUT_DIR" > /dev/null 2>&1
        else
            # Manual loop for old versions that don't support -d
            for f in "$TEMP_TEST_DIR"/*.afp; do
                fname=$(basename "$f")
                java -jar "$jar" "$f" "$OUTPUT_DIR/$fname.xml" > /dev/null 2>&1
            done
        fi
    done
    END_TIME=$(date +%s%3N)

    TOTAL_TIME=$((END_TIME - START_TIME))

    # Manual division
    AVG_TIME_INT=$((TOTAL_TIME / ITERATIONS))
    AVG_TIME_DEC=$(( (TOTAL_TIME % ITERATIONS) ))
    AVG_TIME="${AVG_TIME_INT}.${AVG_TIME_DEC}"

    # Trim leading space from flags
    FLAGS_DISPLAY=$(echo "$flags" | xargs)
    if [ -z "$FLAGS_DISPLAY" ]; then FLAGS_DISPLAY="(None)"; fi

    echo "| **$rel** | \`$FLAGS_DISPLAY\` | $TOTAL_TIME ms | $AVG_TIME ms |"
}

for rel in "${RELEASES[@]}"; do
    git checkout "$rel" --quiet
    ./gradlew shadowJar --quiet > /dev/null 2>&1
    JAR_FILE=$(ls build/libs/*.jar | head -n 1)

    if [ -z "$JAR_FILE" ]; then
        echo "| **$rel** | Build Failed | N/A | N/A |"
        continue
    fi

    # Detect supported flags
    HELP_OUTPUT=$(java -jar "$JAR_FILE" -h 2>&1)

    BASE_FLAGS=""
    if echo "$HELP_OUTPUT" | grep -q -- "-p"; then BASE_FLAGS="$BASE_FLAGS -p"; fi
    if echo "$HELP_OUTPUT" | grep -q -- "-P"; then BASE_FLAGS="$BASE_FLAGS -P"; fi
    if echo "$HELP_OUTPUT" | grep -q -- "-c"; then BASE_FLAGS="$BASE_FLAGS -c"; fi

    if echo "$HELP_OUTPUT" | grep -q -- "-a"; then
        # Run with -a
        run_benchmark "$rel" "$BASE_FLAGS -a" "$HELP_OUTPUT" "$JAR_FILE"
        # Run without -a
        run_benchmark "$rel" "$BASE_FLAGS" "$HELP_OUTPUT" "$JAR_FILE"
    else
        # Run with whatever is supported
        run_benchmark "$rel" "$BASE_FLAGS" "$HELP_OUTPUT" "$JAR_FILE"
    fi
done

rm -rf "$TEMP_TEST_DIR"
git checkout "$INITIAL_BRANCH" --quiet
git stash pop --quiet
```
