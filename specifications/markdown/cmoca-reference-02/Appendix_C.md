# Appendix C. Compliance With Color Management Object Content Architecture

In order to claim that the baseline Color Management Object Content Architecture is supported, a device is required to support certain CMRs as indicated in the following table.

### Table 52. CMR Architecture Compliance Requirements

| CMR Type | Required for Baseline Support? |
| :--- | :--- |
| Halftone | No |
| Tone Transfer Curve | No |
| Color Conversion | Yes |
| Link Color Conversion | No |
| Indexed | No |

In order to claim that the baseline Color Management Object Content Architecture is supported, a device is required to support generic CMRs and device defaults as specified in the architecture reference.

## Glossary

*   **audit CMR:** A color management resource that reflects processing that has been done on an object.
*   **big endian:** A format for storage or transmission of binary data in which the most significant bit (or byte) is placed first.
*   **CIELAB color space:** Internationally accepted color space model used as a standard to define color within the graphic arts industry.
*   **color conversion:** The process of converting colors from one color space to another.
*   **instruction CMR:** A color management resource that identifies processing that is to be done to an object.
*   **PCS (Profile Connection Space):** The reference color space defined by ICC.
